
# UI.md

# FlowSense UI/UX Specification

## Design Philosophy
FlowSense should feel like a modern desktop productivity application inspired by Raycast, Linear, Notion, and Arc Browser.

Principles:
- Fast
- Minimal
- Information-rich
- Dark mode first
- Keyboard friendly
- Responsive layouts

---

# Technology

- React
- TypeScript
- TailwindCSS
- shadcn/ui
- Framer Motion
- Recharts
- React Query

---

# Global Layout

Sidebar
Main Content
Right Context Panel (optional)

Sidebar Items:
- Dashboard
- Timeline
- Workflows
- Suggestions
- Analytics
- Search
- Settings

Persistent Top Bar:
- Search
- Notifications
- Sync Status (Local)
- Theme Toggle

---

# Dashboard

Purpose:
Provide an overview of today's activity.

Widgets:
- Today's Active Time
- Most Used Applications
- Total App Switches
- Detected Workflows
- Pending Suggestions
- Recent Activity Feed

Charts:
- Hourly Activity
- App Usage Pie Chart

Quick Actions:
- View Timeline
- Detect Workflows
- Export Data

Empty State:
"Your activity will appear here once FlowSense starts monitoring."

---

# Timeline

Chronological activity stream.

Each card displays:
- Time
- App icon
- Application name
- Window title
- URL (if browser)

Features:
- Infinite scroll
- Filter by app
- Date picker
- Search
- Jump to timestamp

---

# Workflows

Displays all detected repeated workflows.

Each card contains:
- AI generated name
- Confidence
- Frequency
- Last detected
- Expand button

Expanded View:
- Ordered steps
- Estimated time saved
- Suggested automation

Actions:
- Accept
- Dismiss
- Rename

---

# Suggestions

Cards generated by AI.

Display:
- Title
- Explanation
- Why detected
- Confidence
- Buttons:
  - Accept
  - Dismiss
  - Learn More

# Accessibility

- Screen reader labels
- WCAG AA colors
- Focus indicators
- High contrast mode

---

# Responsive Rules

Desktop first.

Minimum supported width:
1280px

Dark

# Loading States

Every page requires:

# Error Screens

Friendly messages.

Never expose stack traces.

---

# Keyboard Shortcuts

Ctrl+K Search

Ctrl+R Refresh

Ctrl+, Settings

Esc Close dialogs

---

# Implementation Rules

- Components must be reusable.
- No inline styling.
- Maintain consistent spacing.
- Use design tokens.
- Keep pages modular.

Commit each major screen separately and push using the GitHub account already authenticated on the development machine.

---

# Definition of Done

- Every page implemented
- Responsive layouts
- Animations complete
- Accessibility checks passed
- Dark and light themes working
- No visual inconsistencies
